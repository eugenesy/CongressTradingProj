# Project Overview

This project is a Python-based financial data analysis workflow. Its primary purpose is to process and analyze financial transaction data, specifically focusing on congressional trading activities. The workflow involves several steps, from downloading historical stock data to enriching transaction records with various financial metrics and ultimately preparing the data for further analysis, potentially involving embedding techniques.

## Key Components:

*   **`scripts/data_processing/download_tickers.py`**: Downloads historical stock data for all unique tickers found in the transaction data. It saves this data into a `.pkl` file and individual `.parquet` files.
*   **`scripts/data_processing/download_spy.py`**: Downloads historical data for the S&P 500 index (SPY).
*   **`scripts/data_processing/add_spy_columns.py`**: Integrates SPY-related financial metrics into the existing dataset.
*   **`scripts/data_processing/add_closing_prices.py`**: Enriches transaction records by adding the closing prices of the respective tickers on the transaction dates.
*   **`scripts/data_processing/add_excess_returns.py`**: Calculates and appends excess returns to the transaction data, likely comparing individual stock performance against a benchmark (e.g., SPY).
*   **`scripts/data_processing/add_trading_labels.py`**: Adds labels related to trading activities, which could be used for classification or further analysis.
*   **`scripts/data_processing/clean_data.py`**: A dedicated script for cleaning and preprocessing the financial data.
*   **`scripts/data_processing/add_transaction_ids.py`**: Assigns unique transaction IDs and standardizes various data fields for consistency.
*   **`scripts/data_processing/embedding_enhanced.py`**: Performs embedding enhancement on the processed data, likely for machine learning applications or advanced analytical techniques.
*   **`scripts/comparison/compare_data.py`**: A new script to compare the initial processed data (`v5_transactions_with_approp_ticker.csv`) with the final processed data (`v9_transactions.csv`), generating a comprehensive report and visualizations.
*   **`scripts/main.py`**: Orchestrates the entire data processing workflow by calling the functions from the `data_processing` module in the correct sequence. It now includes checks to skip steps if their output files already exist.
*   **`scripts/utils.py`**: Contains common utility functions such as checkpointing and file loading/saving, used across various data processing scripts.

## Technologies Used:

*   **Python**: The primary programming language for all scripts.
*   **Pandas**: Extensively used for data manipulation, analysis, and handling DataFrames.
*   **Pickle**: Used for serializing and deserializing Python objects, particularly for saving and loading large data structures like historical stock data.
*   **tqdm**: Provides fast, extensible progress bars for loops, enhancing user experience during long-running data processing tasks.
*   **os**: Used for interacting with the operating system, such as managing file paths and checking for file existence.
*   **datetime**: Utilized for handling and manipulating date and time information, crucial for time-series financial data.
*   **Sentence-Transformers**: Used for generating embeddings from text data.
*   **Matplotlib & Seaborn**: Used for data visualization in the comparison report.

## Data Files:

*   **`data/all_tickers_historical_data.pkl`**: A pickled Python object (likely a dictionary of Pandas DataFrames) containing historical stock data for all unique tickers identified in the transaction records.
*   **`data/spy_historical_data.pkl`**: A pickled Python object storing the historical data for the S&P 500 index (SPY).
*   **`data/congress-trading-all-reduced-attributes.csv`**: A raw CSV file containing initial congressional trading transaction data with various attributes.
*   **`data/v5_transactions_with_approp_ticker.csv`**: A CSV file, likely a pre-processed version of the raw transaction data, where tickers have been standardized or validated. This file serves as input for downloading historical data.

*   **`data/comparison_results/` directory**: Contains the generated comparison report and visualizations.

## Development Conventions:

*   **Modular Design**: The workflow is broken down into smaller, focused Python files, each containing a specific function for a processing step.
*   **Orchestration**: A central `main.py` script manages the execution order of these modular functions, with added idempotency checks to skip already completed steps.
*   **Utility Functions**: Common functionalities are consolidated into `utils.py` for reusability and consistency.
*   **Configuration**: Key parameters and file paths are defined as constants within each module or passed as arguments, improving configurability.
*   **User Feedback**: Progress bars (`tqdm`) are integrated into long-running operations to provide visual feedback on the script's progress.
*   **Error Reporting**: Mechanisms for reporting failed operations (e.g., `failed_tickers_report.txt`) are included to assist in debugging and data validation.
